SQL Lab · duckdb-wasm

Run SQL against the settlement panel in your browser

The full disputed sample lives in a small DuckDB file (10 MB) that loads into your browser via WebAssembly. "Lab" because it runs locally over a static snapshot, not over live market data. No server. No network roundtrip after the initial load. Tables: contracts (2,221), episodes (2,668), funnel, chain_signatures, flip_decomposition.

Schema reference

contracts (2,221 rows)
  condition_id          VARCHAR   keccak256 hash, internal
  first_proposal        VARCHAR   'Yes' / 'No' / 'Other'
  final_payoff          VARCHAR   'Yes' / 'No' / 'Other' / 'Invalid' / 'Pending'
  revised               BOOLEAN   final_payoff != first_proposal
  multi_episode         BOOLEAN
  chain_type            VARCHAR   'Request-voiding chain' or 'Repeated adapter-routed request'
  question_text         VARCHAR   nullable (79.1% coverage)
  category              VARCHAR   nullable
  description           VARCHAR   nullable, fuller resolution criteria
  episode_sequence      VARCHAR   raw "1:Yes->clean_yes(D)|2:..." encoding
  S_i                   VARCHAR   raw paper variable
  terminal_state_type   VARCHAR   raw paper variable
  first_proposal_time   BIGINT    POSIX seconds
  final_settlement_time BIGINT    POSIX seconds

episodes (2,668 rows)
  condition_id     VARCHAR   FK to contracts
  episode_id       VARCHAR
  t_proposal       BIGINT
  t_dispute        BIGINT    nullable
  t_final          BIGINT    nullable
  proposed         VARCHAR
  final_outcome    VARCHAR
  was_disputed     BOOLEAN
  outcome_flipped  BOOLEAN

funnel              -- 9-step settlement funnel ladder (CSV import)
chain_signatures    -- 23 raw chain-pattern aggregates with category
flip_decomposition  -- 7 subset breakdowns of the 530 revisions
