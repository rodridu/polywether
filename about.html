About

What this monitor covers

A settlement-risk layer for prediction markets, built on a panel of disputed Polymarket contracts resolved through UMA. Track rule clarity and historical revision patterns before quoting or trading.

Coverage

  • 661,949 Polymarket-via-UMA settlement conditions tracked.
  • 2,221 disputed contracts (≥1 counter-bond posted) — analysis sample.
  • 530 (24.1%) end with a final payoff different from the first proposal.
  • 64 candidate rule-divergent revisions flagged by a two-pass LLM-coded benchmark (paper headline), concentrated in Path C request-voiding chains. Preliminary pending independent blinded re-coding.

What we do NOT claim

  • The unconditional rate at which a random market receives an erroneous final payoff.
  • Whether any specific contract's final payoff is correct in real-world terms.
  • UMA tokenholder voter identity, vote concentration, or turnout.
  • Causal effects of any UMA policy change.

Data source

Polymarket markets and UMA Optimistic Oracle resolution events. Aggregates refresh weekly via a GitHub Actions workflow on committed source CSVs; per-contract data and the duckdb-wasm DB refresh manually when the paper's analysis pipeline regenerates the panel.

Researcher

Hongzhen Du, Northwestern University. Built on the working paper "When Facts Aren't Payoffs: The Resolution-Settlement Gap in Prediction Markets." Full methodology, sample construction, and limitations: Research →. Data fields and JSON endpoints: Docs →.
